# iterp

Disclaimer: This package is mostly an experiment and will likely see breaking
changes. There are other packages which fulfill the same purpose and I encourage
you to use them if you are trying to make reliable software.

A consistent set of packages for processing sequences and generic container
types.  This library is inspired by [samber/lo](https://github.com/samber/lo)
but aims to provide a simpler, more consistent interface between analogous
operations on slices, sequences, and maps.
